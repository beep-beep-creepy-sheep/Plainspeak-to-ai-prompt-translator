# Mode Templates

## Coding

Include:

- Goal
- User flow
- Inputs and outputs
- File or component expectations
- Data model or state model
- Edge cases
- Tests and manual verification
- Deployment target, if known

Final prompt should be implementation-ready for Codex.

## Writing

Include:

- Audience
- Purpose
- Tone
- Length
- Structure
- Claims that need support
- Call to action, if relevant

## Analysis

Include:

- Decision context
- Evidence needed
- Data sensitivity
- Definitions
- Comparison baseline
- Caveats
- Recommended output format

Avoid investment, legal, medical, or regulated advice unless the user explicitly asks for a safe, non-advisory educational framing.

## AI Risk

Include:

- Use case
- Stakeholders
- Failure modes
- Misuse risks
- Privacy risks
- Human review points
- Escalation criteria
- Safe-use checklist

## Evaluation

Include:

- Eval suite goal
- Normal case
- Edge case
- Unsafe input case
- Expected behavior
- Scoring rubric
- Human review notes

Make pass/fail criteria observable.
